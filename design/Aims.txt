Aims
====

Main goals
----------
- Provide a website where non-technical users can easily create individual web pages or whole sites
- Allow users to build interesting and good-looking pages or applications by combining web components
- Very low running costs so the basic service can be free
- Still useful for large sites and skilled developers

What you might do with it
-------------------------
- In 10 minutes: a party organiser puts up a page describing the event and emails a link to the guests
- In 1 hour: a language teacher creates a mini-app for students to learn and test themselves on each week's new vocabulary
- In 8 hours: a teenager creates his own adventure game
- In 40 hours: an accountant produces an online app for her clients to enter their bookkeeping directly


High level requirements
-----------------------
- Provide a browser front-end for a CMS
- Wrap in a web component that can be included in any page
- Uses REST API for all loading, saving, file lists
- UX familiar to users of file finders and word processors
- Can manage all types of files - HTML, CSS, image, FreeSheet, JSON
- Edit HTML in WYSIWYG
- Make viewing/previewing easy as well as editing
- Support preview/publish
- Support file permissions - editable or not
- Easy to navigate file tree
- Multiple files open simultaneously

HTML specific
-------------
- Can use web components as easily as any other element in the HTML editor
- Can use non-UI web components
- Can configure web components easily
- Can connect web components to build an app in the page




Nice to have
------------
- Sub-components eg file list, editor could be used separately
- Sub-components could be replaced eg interface to REST API, editor
